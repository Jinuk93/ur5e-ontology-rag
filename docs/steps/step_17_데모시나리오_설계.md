## 2. 목표

- “온톨로지 기반 추론의 가치”를 **3가지 데모 시나리오**로 재현 가능하게 고정한다.
- 데모는 UI(Next.js)에서 보여주되, 백엔드 API만으로도 같은 내용을 재현할 수 있어야 한다.

---

## 3. 데모 전제(실행)

### 3.1 백엔드

- `python scripts/run_api.py --host 127.0.0.1 --port 8002`

### 3.2 프론트(선택, UI 데모)

- `cd frontend`
- `npm run dev`
- 필요 시 `.env.local`:
  - `NEXT_PUBLIC_API_URL=http://127.0.0.1:8002`

---

## 4. 시나리오

### 시나리오 1: 온톨로지 추론 + 근거(Trace)

- 질문 예: “Fz가 -350N인데 이게 뭐야?”
- 기대:
  - 상태/패턴/원인/권장 조치가 구조화되어 제시
  - `trace_id`가 발급되고 Evidence Drawer(또는 `/api/evidence/{trace_id}`)로 근거 확인 가능

### 시나리오 2: 관계 탐색(그래프/경로)

- 질문 예: “왜 이렇게 됐어?” 또는 원인/에러코드 중심 질문
- 기대:
  - 온톨로지 경로(노드/관계 배열)가 노출
  - Graph View에서 경로/서브그래프 탐색

### 시나리오 3: 센서 기반 Live 데모(REST+SSE)

- 기대:
  - Live View가 센서 데이터를 폴링 또는 SSE로 수신
  - 이벤트/패턴이 표시되며, 데이터 부재 시에도 앱이 죽지 않고 degrade(빈 목록/에러 이벤트)한다.

---

## 5. 완료 기준(Definition of Done)

- 3개 시나리오가 “명령어 + 질문 + 확인 포인트” 형태로 문서화되어 있다.
- Step 16의 검증 루틴(E2E)이 PASS인 환경에서, 데모 시나리오도 그대로 재현 가능하다.
