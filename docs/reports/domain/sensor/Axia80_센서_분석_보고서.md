# 1. 센서란 무엇인가?

## 1.1 센서의 기본 개념

센서(Sensor)는 **물리적인 현상을 전기 신호로 변환하는 장치**입니다.

```
물리적 현상          센서           전기 신호         컴퓨터
(힘, 온도, 빛)  →  [변환]  →  (전압, 전류)  →  [숫자로 표시]

예시:
- 체중계: 무게(힘) → 숫자(kg)
- 온도계: 온도 → 숫자(°C)
- Axia80: 힘/토크 → 숫자(N, Nm)
```

## 1.2 왜 로봇에 센서가 필요한가?

로봇은 기본적으로 **"눈이 멀고 감각이 없는"** 기계입니다.

```
센서 없는 로봇:
┌─────────────────────────────────────────┐
│  "명령받은 위치로 이동해!"               │
│       ↓                                 │
│  장애물이 있어도 그냥 부딪힘             │
│  물건을 너무 세게 잡아서 부숨            │
│  조립이 안 됐는데 계속 밀어붙임           │
└─────────────────────────────────────────┘

센서 있는 로봇:
┌─────────────────────────────────────────┐
│  "명령받은 위치로 이동해!"               │
│       ↓                                 │
│  "어? 뭔가 힘이 느껴지네?"              │
│       ↓                                 │
│  "멈추고 상황 판단하자"                  │
└─────────────────────────────────────────┘
```

**Force/Torque 센서는 로봇에게 "촉각"을 부여합니다.**

# 2. Axia80 센서 소개

## 2.1 제품 개요

| 항목 | 내용 |
|------|------|
| **제조사** | ATI Industrial Automation (미국) |
| **제품명** | Axia80 |
| **유형** | 6축 Force/Torque 센서 |
| **용도** | 산업용 로봇의 힘/토크 측정 |
| **가격대** | ATI 제품 중 저가형 (약 $2,000~3,000) |

## 2.2 물리적 특성

```
        ┌─────────────────┐
        │     위 (로봇)    │  ← 로봇 팔에 연결
        ├─────────────────┤
        │                 │
        │    ◯ Axia80 ◯   │  직경: 82mm (약 8cm)
        │                 │  높이: 25.4mm (약 2.5cm)
        │                 │  무게: 0.28~0.68kg
        ├─────────────────┤
        │    아래 (툴)     │  ← 그리퍼/공구에 연결
        └─────────────────┘
```

## 2.3 왜 "Axia80"인가?

- **Axia**: "축(Axis)"을 의미 - 여러 축을 측정
- **80**: 직경 약 80mm를 의미

## 2.4 모델 종류

Axia80은 측정 범위에 따라 여러 모델이 있습니다:

| 모델 | Fx, Fy 범위 | Fz 범위 | 용도 |
|------|------------|---------|------|
| **M8** | ±75N ~ ±150N | ±235N ~ ±470N | 가벼운 작업 |
| **M20** | ±200N ~ ±400N | ±600N ~ ±1200N | 중간 작업 |
| **M50** | ±500N ~ ±1000N | ±1500N ~ ±3000N | 무거운 작업 |

# 3. 6축의 의미 - 힘과 토크

## 3.1 힘(Force)과 토크(Torque)의 차이

### 힘 (Force)
**직선 방향으로 밀거나 당기는 것**

```
힘의 예시:

     ──→ Fx (밀기)          ↓ Fz (누르기)

     책상 위 물건을           의자에 앉을 때
     옆으로 밀 때             체중이 누르는 것
```

- 단위: **N (뉴턴)**
- 1N ≈ 100g 물체에 작용하는 중력
- 1kg 물체 = 약 9.8N ≈ 10N

### 토크 (Torque)
**회전시키려는 힘 (비트는 힘)**

```
토크의 예시:

     ↺ Tz (비틀기)          ↻ Tx, Ty (기울이기)

     병뚜껑 돌릴 때          문 손잡이 누를 때
     드라이버 돌릴 때         시소 기울어질 때
```

- 단위: **Nm (뉴턴미터)**
- 1Nm = 1m 떨어진 곳에서 1N의 힘으로 돌리는 것

## 3.2 3차원 공간의 6가지 움직임

3차원 공간에서 물체는 **6가지 방향**으로 움직일 수 있습니다:

```
              Z (위/아래)
              ↑
              │    Y (앞/뒤)
              │   /
              │  /
              │ /
              ┼─────────→ X (좌/우)
             /│
            / │

3가지 직선 이동 (Force):        3가지 회전 (Torque):
- X축 이동 → Fx                - X축 기준 회전 → Tx (Roll)
- Y축 이동 → Fy                - Y축 기준 회전 → Ty (Pitch)
- Z축 이동 → Fz                - Z축 기준 회전 → Tz (Yaw)
```

## 3.3 비행기로 이해하는 6축

```
                     Tz (Yaw) - 좌우 방향 전환
                         ↺
                         │
                   ┌─────┴─────┐
                   │  ✈️ 비행기  │
         Tx (Roll) ↻│           │↺ Ty (Pitch)
         기체 기울임 │           │ 기수 상하
                   └───────────┘
                         │
                    Fz (상승/하강)

        Fx (전진/후진)  ←──→  Fy (좌/우 이동)
```

# 4. 각 축별 상세 설명

## 4.1 Fx (X축 힘)

```
        로봇 팔
           │
           ▼
      ┌─────────┐
      │ Axia80  │ ←──── Fx: 이 방향 힘
      └─────────┘
           │
         [툴]
```

| 항목 | 내용 |
|------|------|
| **측정 대상** | 수평면에서 X 방향(전후)으로 작용하는 힘 |
| **측정 범위** | ±75N ~ ±1000N (모델에 따라) |
| **해상도** | 0.04N (약 4g 무게 차이 감지 가능) |
| **양수 (+)** | 센서 기준 +X 방향으로 힘이 작용 |
| **음수 (-)** | 센서 기준 -X 방향으로 힘이 작용 |

### 언제 Fx가 발생하나?
- 로봇이 물체를 **옆으로 밀 때**
- **측면 충돌**이 발생했을 때
- 조립 시 부품이 **옆으로 밀릴 때**

## 4.2 Fy (Y축 힘)

| 항목 | 내용 |
|------|------|
| **측정 대상** | 수평면에서 Y 방향(좌우)으로 작용하는 힘 |
| **특성** | Fx와 유사하지만 90° 다른 방향 |

### 언제 Fy가 발생하나?
- Fx와 유사한 상황, 방향만 다름
- Fx와 Fy를 함께 보면 **수평면에서의 힘 방향**을 알 수 있음

```
          Fy (+)
            ↑
            │
  Fx (-) ←──┼──→ Fx (+)
            │
            ↓
          Fy (-)

  예: Fx=10N, Fy=10N이면
      대각선 방향으로 약 14N의 힘
```

## 4.3 Fz (Z축 힘) ⭐ 가장 중요

```
        로봇 팔
           │
           ▼
      ┌─────────┐
      │ Axia80  │
      └─────────┘
           │
           ▼ Fz: 이 방향 힘 (누르는/당기는)
         [툴]
           │
           ▼
        [물체]
```

| 항목 | 내용 |
|------|------|
| **측정 대상** | 수직 방향(위아래)으로 작용하는 힘 |
| **측정 범위** | ±235N ~ ±3000N (모델에 따라) |
| **해상도** | 0.04N |
| **양수 (+)** | 센서를 당기는 힘 (위로) |
| **음수 (-)** | 센서를 누르는 힘 (아래로) |

### 왜 Fz가 가장 중요한가?

1. **중력의 영향**
   - 툴과 물체의 무게가 항상 -Fz로 작용
   - 기본값 = -(툴 무게 + 물체 무게) × 9.8

2. **작업의 대부분이 Z축**
   - 물건 집기: Fz 변화로 감지
   - 물건 놓기: Fz 변화로 감지
   - 누르기/조립: Fz로 힘 조절

3. **충돌 감지의 핵심**
   - 바닥/물체에 부딪히면 Fz 급증

### Fz 값의 해석

| Fz 값 | 의미 |
|-------|------|
| **0N** | 아무것도 들고 있지 않음 (센서만) |
| **-10N** | 약 1kg 물체를 들고 있음 |
| **-50N** | 약 5kg 물체를 들고 있음 (UR5e 최대 페이로드) |
| **-100N** | 물체를 세게 누르고 있음 |
| **-300N 이상** | 과부하 또는 충돌 의심 |

## 4.4 Tx (X축 토크, Roll)

```
            │
      ┌─────┴─────┐
      │   센서    │
      └───────────┘
       ↙         ↘
      ↺    ←    ↺

  X축을 기준으로 회전 (기울어짐)
```

| 항목 | 내용 |
|------|------|
| **측정 대상** | X축 기준 회전력 (좌우 기울어짐) |
| **측정 범위** | ±4Nm ~ ±8Nm |
| **해상도** | 0.002Nm |

### 언제 Tx가 발생하나?
- 툴이 한쪽으로 **기울어질 때**
- **비대칭 물체**를 잡았을 때
- 충돌로 인해 **옆으로 밀릴 때**

## 4.5 Ty (Y축 토크, Pitch)

| 항목 | 내용 |
|------|------|
| **측정 대상** | Y축 기준 회전력 (앞뒤 기울어짐) |
| **특성** | Tx와 유사, 90° 다른 방향 |

### Tx와 Ty의 조합
```
             Ty (+)
              ↑
              │
   Tx (-) ←───┼───→ Tx (+)
              │
              ↓
             Ty (-)

Tx와 Ty를 함께 보면 → 어느 방향으로 기울어졌는지 알 수 있음
```

## 4.6 Tz (Z축 토크, Yaw)

```
         ↺ Tz
        ┌───┐
        │   │
        └───┘

  Z축을 기준으로 회전 (비틀림)
```

| 항목 | 내용 |
|------|------|
| **측정 대상** | Z축 기준 회전력 (비틀림) |
| **측정 범위** | ±4Nm ~ ±8Nm |

### 언제 Tz가 발생하나?
- **나사 조이기/풀기** 작업
- **뚜껑 돌리기** 작업
- 물체가 **회전 저항**을 줄 때
- **비틀림 충돌**이 발생했을 때

# 5. 측정값의 실제 의미

## 5.1 정상 상태 (Idle)

로봇이 아무 작업도 하지 않을 때:

```
상황: 로봇 팔 끝에 1kg 그리퍼 장착, 물건 없음

예상 측정값:
┌────────┬─────────┬──────────────────────┐
│ 축     │ 값      │ 이유                  │
├────────┼─────────┼──────────────────────┤
│ Fx     │ ≈ 0N    │ 수평 힘 없음          │
│ Fy     │ ≈ 0N    │ 수평 힘 없음          │
│ Fz     │ ≈ -10N  │ 그리퍼 무게 (1kg)     │
│ Tx     │ ≈ 0Nm   │ 기울어짐 없음         │
│ Ty     │ ≈ 0Nm   │ 기울어짐 없음         │
│ Tz     │ ≈ 0Nm   │ 비틀림 없음           │
└────────┴─────────┴──────────────────────┘
```

## 5.2 물건 집기 (Pick)

```
시간 흐름 →

Fz
 │
 │  ①        ②         ③
-10├───────────────────────  그리퍼만
   │         ↓
-50├─────────┴───────────── 물건 집음 (4kg)
   │
   └────────────────────────→ 시간

① 접근 중 (Fz = -10N, 그리퍼만)
② 물건 접촉/집기 (Fz 급격히 변화)
③ 들어올림 (Fz = -50N, 그리퍼 + 물건)
```

## 5.3 충돌 (Collision)

```
시간 흐름 →

Fz
 │
 │              ②
-50├─────────────┬───────  정상 작업
   │             │↓ 급격한 변화
   │             │
-200├────────────┴───────  충돌!
   │
   └────────────────────→ 시간

특징:
- 매우 짧은 시간에 (< 100ms)
- 큰 힘 변화 (100N 이상)
- 여러 축 동시 변화 (Fx, Fy, Fz)
```

## 5.4 과부하 (Overload)

```
시간 흐름 →

Fz
 │
 │  정상      과부하 상태
-50├──────┐
   │      │
   │      └───────────────  장시간 지속
   │
-150├──────────────────────
   │
   └────────────────────→ 시간

특징:
- 급격하지 않지만
- 정상 범위를 벗어난 값이
- 장시간 지속됨 (> 5초)
```

## 5.5 진동 (Vibration)

```
시간 흐름 →

Fz
 │     ∿∿∿∿∿∿∿∿∿∿∿∿
-50├───∿              ∿───  고주파 진동
   │
   └────────────────────→ 시간

특징:
- 빠른 주기로 (> 5Hz)
- 값이 위아래로 흔들림
- 기계적 불안정 또는 마모 의심
```

```

      Z (위)                          Z (로봇→툴 방향)

      │                               │

센서는 Tool Flange에 장착되므로,
```

## 6.4 Axia80 측정값과 UR5e 상태의 관계

| Axia80 측정 | UR5e 관점에서의 의미 |
|-------------|---------------------|
| **Fz 증가** | 툴이 물체를 누르고 있음, 또는 물체를 들어올림 |
| **Fz 급증** | 충돌 가능성, 또는 단단한 표면에 접촉 |
| **Fx, Fy 변화** | 수평 방향 힘 발생, 측면 접촉/충돌 |
| **Tx, Ty 변화** | 툴이 기울어짐, 비대칭 하중 |
| **Tz 변화** | 비틀림 발생, 나사 조임/회전 저항 |

## 6.5 UR5e 에러코드와 센서 데이터의 연결

| 센서 패턴 | 관련 가능한 UR5e 상태/에러 |
|----------|--------------------------|
| Fz 급격한 스파이크 | Protective Stop (C153), Safety Stop |
| 지속적 고하중 | Joint Overload, Payload 초과 경고 |
| 고주파 진동 | Joint 마모, 기계적 문제 |
| Baseline 드리프트 | 센서 캘리브레이션 필요, 온도 변화 |

# 7. 실제 현장에서의 활용

## 7.1 대표적인 응용 분야

### 1) 조립 (Assembly)

```
시나리오: 부품을 구멍에 삽입

정상 과정:
1. 접근 (Fz ≈ -50N, 부품 무게)
2. 접촉 (Fz 약간 증가)
3. 삽입 중 (Fz 점진적 증가, 마찰)
4. 완전 삽입 (Fz 또는 Tz 스파이크 - "바닥에 닿음")

문제 감지:
- 삽입 중 Fx, Fy 증가 → 정렬 불량
- 예상보다 빠른 Fz 스파이크 → 부품 걸림
- Tz 과도 → 나사산 손상 위험
```

### 2) 폴리싱/연마 (Polishing)

```
시나리오: 표면을 일정한 힘으로 연마

목표: Fz를 일정하게 유지 (예: -20N)

정상 과정:
- 표면 굴곡에 따라 로봇이 위치 조절
- Fz는 항상 -20N 유지
- 표면을 따라 이동

문제 감지:
- Fz 급증 → 표면 돌출부 또는 이물질
- Fz 급감 → 표면에서 떨어짐
- 진동 → 불안정한 연마
```

### 3) 픽 앤 플레이스 (Pick & Place)

```
시나리오: 물건을 집어서 옮기기

정상 과정:
1. 접근 (Fz = 그리퍼 무게)
2. 집기 (Fz 변화 = 물건 무게 추가)
3. 이동 (Fz 유지)
4. 놓기 (Fz 변화 = 물건 무게 감소)

문제 감지:
- 집기 후 Fz 변화 없음 → 물건 못 잡음
- 이동 중 Fz 변화 → 물건 떨어짐
- 놓기 전 Fz 급증 → 충돌
```

### 4) 충돌 감지 (Collision Detection)

```
목적: 사람이나 장애물과의 충돌 시 즉시 정지

감지 방법:
- 짧은 시간(< 50ms)에
- 큰 힘 변화(예: > 50N)가 발생하면
- 즉시 로봇 정지

임계값 예시:
- 협동로봇: 매우 민감 (낮은 임계값)
- 산업로봇: 덜 민감 (높은 임계값)
```

## 7.2 센서 없이 vs 센서 있이

| 작업 | 센서 없이 | 센서 있이 |
|------|----------|----------|
| 조립 | 위치만 제어, 힘 조절 불가 | 힘 기반 조립, 불량 감지 |
| 연마 | 일정 경로만, 힘 변동 | 일정한 힘 유지, 품질 향상 |
| 충돌 대응 | 충돌해도 계속 진행 | 즉시 감지하고 정지 |
| 품질 검사 | 별도 장비 필요 | 조립 중 실시간 검사 |

# 8. 이상 상태 판단 기준

## 8.1 임계값 설정의 기본 원칙

```
                    Critical (즉시 정지)
    ──────────────────────────────────────  예: 500N

                    Warning (주의/경고)
    ──────────────────────────────────────  예: 200N

                    Normal (정상 작업 범위)
    ──────────────────────────────────────  예: 0~100N
```

## 8.2 패턴별 판단 기준 (예시)

### 충돌 (Collision)

| 조건 | 값 |
|------|-----|
| 힘 변화량 | > 100N |
| 시간 | < 100ms |
| 영향 축 | Fx, Fy, Fz 중 하나 이상 |
| 대응 | 즉시 정지, 위치 확인 |

### 과부하 (Overload)

| 조건 | 값 |
|------|-----|
| Fz 절대값 | > 200N (5kg 페이로드 기준) |
| 지속 시간 | > 5초 |
| 대응 | 경고, 페이로드 확인 |

### 진동 (Vibration)

| 조건 | 값 |
|------|-----|
| 주파수 | > 5Hz |
| 진폭 | 정상의 2배 이상 |
| 지속 시간 | > 10초 |
| 대응 | 경고, 기계 점검 |

### 드리프트 (Drift)

| 조건 | 값 |
|------|-----|
| 기준값 변화 | > 10% |
| 시간 | > 30분 |
| 대응 | 캘리브레이션 권장 |

## 8.3 복합 패턴

실제로는 여러 패턴이 **동시에** 또는 **연속으로** 발생할 수 있습니다:

```
예: 조립 실패 시나리오

시간 →
1. 정상 접근
2. 약한 충돌 (부품 걸림)
3. 계속 밀어붙임 (과부하)
4. 기계적 손상 (진동 발생)

센서 데이터:
Fz: ────┐  ┌───────────────∿∿∿
        └──┘
     정상  충돌  과부하    진동
```

# 9. 우리 시스템에서의 적용

## 9.1 현재 시뮬레이션 데이터의 문제점

우리가 만든 가짜 데이터:

```python
# 현재 설정 (scripts/generate_sensor_data.py)
Fz 정상값: -5N
Fz 충돌: 150N
Fz 과부하: 200N
```

**문제점:**

| 항목 | 현재 | 실제 | 문제 |
|------|------|------|------|
| Fz 정상값 | -5N | -10N ~ -50N | 너무 작음 (거의 빈 상태) |
| 충돌 임계값 | 150N | 모델/작업에 따라 다름 | 맥락 없이 하드코딩 |
| 샘플링 레이트 | 1Hz | 최대 8000Hz | 너무 느림 |

## 9.2 올바른 온톨로지를 위해 필요한 것

### 센서 측면
```yaml
Axia80_Sensor:
  model: "M8" 또는 "M20" 등 명시
  calibration: "SI-75-4" 또는 "SI-150-8" 명시
  mounting: "UR5e Tool Flange"
  coordinate_system: "Tool Center Point 기준"

  axes:
    Fz:
      unit: "N"
      range: [-235, 235]  # SI-75-4 기준
      resolution: 0.04
      normal_idle: -10  # 그리퍼만
      normal_with_5kg: -60  # 그리퍼 + 5kg 물체
      warning_threshold: 150
      critical_threshold: 200
```

### UR5e 측면
```yaml
UR5e:
  payload_max: 5kg (≈ 50N)
  joints: [J0, J1, J2, J3, J4, J5]

  error_codes:
    C153:
      trigger: "Position deviation"
      related_sensor: "Fz 급증, 충돌 패턴"
    C189:
      trigger: "Joint overload"
      related_sensor: "Fz 지속적 고값"
```

### 관계 정의
```yaml
relationships:
  - Axia80 MOUNTED_ON UR5e.ToolFlange
  - Axia80.Fz INDICATES payload_weight
  - Axia80.Fz_spike INDICATES collision
  - collision MAY_TRIGGER C153
  - C153 HAS_RESOLUTION "Check robot path"
  - "Check robot path" DOCUMENTED_IN "Service Manual p.XX"
```
